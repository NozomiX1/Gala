# Domain Docs

This is a single-context repo.

## Before exploring

Read these if they exist:

- `CONTEXT.md` at the repo root
- `docs/adr/`

If they do not exist, proceed silently. Do not create them unless the user asks or a skill specifically needs to capture domain language or architectural decisions.

## Current domain sources

- `AGENTS.md` is the canonical operational guide for agents.
- `README.md` documents current user-facing behavior.
- `CHANGELOG.md` records shipped changes.
- `docs/plans/` contains historical design and implementation notes; do not treat old plans as current truth when they conflict with `AGENTS.md`, `README.md`, or code.

## Vocabulary

Use the project's established terms: game library, runtime profile, Wine prefix/bottle, managed dependency, engine detection, local signatures, VNDB fallback, runtime configuration, and migration.
